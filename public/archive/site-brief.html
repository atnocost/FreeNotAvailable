01

Overview

atnocost.cc is the digital home for FREE, an R&B songwriter operating under the OWJV (Other Worldly, Just Visiting) imprint. The site houses the full catalog, film library, fan engagement system, press outreach tools, and token-gated business documents.

What this site is: Not a standard artist website. It's a controlled environment — part portfolio, part distribution system, part gated vault. Every page serves a specific function in a larger rollout strategy. Public-facing pages are for fans and discovery. Internal pages are for us. Token-gated pages are for press and industry, accessed only through unique links we generate and send. The site is the central nervous system for everything FREE does digitally.

Stack: Next.js 16 + Tailwind CSS v4, deployed on Vercel. Domain on Cloudflare (DNS + email routing). Email via Resend. State via Vercel KV. Fan subscribers via Beehiiv (pending integration).

Why this stack: Next.js gives us server-side routes and middleware — that's what powers the token gating and auth without a separate backend. Vercel handles deployment (push to git, site updates automatically). Cloudflare handles the domain and routes reply emails to Gmail. Resend sends outbound email (press pitches, fan notifications). Vercel KV is a Redis database that stores tokens and outreach status. Beehiiv will handle the fan email list once connected — we build the email HTML, Beehiiv handles scheduling and unsubscribes.

Repo: github.com/atnocost/FreeNotAvailable

Repo note: This is our fork with full push access. The upstream repo (ObsidianWRK/FreeNotAvailable) is read-only to us. All work goes to the atnocost fork, which is what Vercel deploys from.

03

Authentication & Gating

Three independent token systems protect different content tiers. All validated in middleware — env vars checked first, then Vercel KV for dynamically generated tokens.

How middleware works: Every request to the site passes through a middleware layer before reaching the page. The middleware checks the URL path and decides: is this a public route? Does it need a token? Does it need the admin password? If access is denied, the visitor gets redirected to the appropriate gate page. This all happens server-side before any page content loads — unauthorized visitors never see protected content, not even for a flash.

Token Gates

  • Ekthesis Gate — Protects /ekthesis. Tokens generated per press contact via /internal/outreach or /internal/tokens. URL format: atnocost.cc/ekthesis?token=XXXX
  • Brief Gate — Protects /brief. Same token system, separate namespace.
  • Site Gate — Optional site-wide lock. Activated by setting SITE_GATE_ENABLED=true. Visitors see email capture, then get access cookie (365 days).
Token lifecycle: Tokens are created at /internal/tokens or automatically when sending a press pitch at /internal/outreach. Each token is a random string stored in Vercel KV (Redis). When a visitor hits a gated URL with ?token=XXXX, the middleware looks it up. If found and valid, the visitor gets through. Tokens persist until manually revoked. We can revoke any token at any time from the tokens page, immediately cutting off access. Each token is tied to a specific gate (ekthesis, brief, or site) — an ekthesis token won't work on /brief and vice versa.

Internal Auth

  • Password login at /internal/login. Sets httpOnly cookie (24hr). Protects all /internal/* routes.
  • Password stored as INTERNAL_AUTH_TOKEN env var on Vercel.
Operational note: The internal password is a single shared password for the team. It's set as an environment variable on Vercel — to change it, update INTERNAL_AUTH_TOKEN in the Vercel dashboard and redeploy. The cookie is httpOnly (can't be read by JavaScript) and expires after 24 hours, so you'll need to re-login daily. If someone leaves the team, change the password immediately.

Bypass Rules

  • All /api/* routes — always allowed
  • Static assets (/images, /fonts, /textures, _next) — always allowed
  • favicon.ico, robots.txt, sitemap.xml — always allowed
Why bypasses matter: Images and fonts must always load — even inside gated pages and emails. If /images/ were gated, the email GIFs wouldn't render in anyone's inbox. API routes are bypassed because they handle their own authentication internally (checking tokens/passwords in the request body or headers, not via cookies).

API Routes

What these are: API routes are server-side endpoints that handle actions — logging in, subscribing, sending emails, managing tokens. They're not pages you visit in a browser. They're called by the frontend (forms, buttons) or by external services. When someone fills out the email capture form, the form sends a POST request to /api/subscribe. When you click "Send Pitch" in the outreach tool, it calls /api/outreach/send.
EndpointMethodPurpose
/api/subscribe POST Fan email signup. Validates email, forwards to freeisavailable@atnocost.cc via Resend. Beehiiv integration pending.
/api/internal-auth POST Admin login. Validates password, sets internal_auth cookie (24hr).
/api/site-auth GET Site gate validator. Checks token, sets site_access cookie (1yr), redirects to destination.
/api/outreach GET/POST/PATCH/DELETE Outreach campaign management. Generate ekthesis tokens, update contact status (pending/sent/bounced/opened), revoke tokens. Stored in Vercel KV.
/api/outreach/send POST Send pitch emails. T1-T3 from free@atnocost.cc (artist voice), T4 from freeisavailable@atnocost.cc (imprint voice). Via Resend.
/api/tokens GET/POST/DELETE General token CRUD for ekthesis, brief, and site gates. Stored in Vercel KV.
The two email addresses: This is important. We send from two different addresses depending on who we're talking to. free@atnocost.cc is the artist's voice — first-person, used for press contacts (Tiers 0-3). freeisavailable@atnocost.cc is the imprint voice — third-person, used for industry contacts (Tier 4: labels, distributors, A&R). Both addresses route replies to the same Gmail inbox (sound.otherworld@gmail.com) via Cloudflare Email Routing. This means we see all replies in one place regardless of which address was used.

Fans aren't being marketed to, they're being invited deeper into the world. Each visit unlocks something: an origin story, a digital booklet, a manuscript. Press Outreach

144 contacts across 5 tiers. Managed at /internal/outreach.

The outreach system: We have a built-in press campaign tool at /internal/outreach. It's not a third-party CRM — it's custom-built into the site. The 144 contacts are stored in the codebase (lib/press-contacts.ts) and organized into 5 tiers based on reach and relationship. Each contact can be assigned a unique ekthesis token, sent a personalized pitch email, and tracked through the pipeline. The outreach tool shows status for every contact: pending, sent, opened, or bounced. This gives us full visibility into our press campaign without any external tools.
  • Tier 0 (VIP) — First access. Exclusive contacts (e.g., Patu Ong). These get personal attention and custom pitches.
  • Tier 1 (Major) — Pitchfork, Rolling Stone senior writers. High-impact outlets, carefully worded pitches.
  • Tier 2 (Mid) — Mid-tier indie publications. Broader reach, still curated.
  • Tier 3 (Indie/Blogs) — Independent music press. Volume play with quality contacts.
  • Tier 4 (Industry) — Labels, distributors, A&R. Business contacts, not editorial. Uses imprint voice, not artist voice.
Tier strategy: Tiers aren't just organization — they determine timing and voice. VIP and T1 get contacted first with the most care. T4 contacts hear from the imprint (freeisavailable@atnocost.cc) in third person because they're business relationships, not editorial. The pitch emails for the top 15 contacts have been individually rewritten with personalized, film-centered language. CMC (Creative Minds Coalition) is a silent partner — they are never mentioned in any outreach or public-facing material.

Email Voices

  • free@atnocost.cc — Artist voice. First-person. Used for T0-T3 (press contacts).
  • freeisavailable@atnocost.cc — Imprint voice. Third-person. Used for T4 (industry contacts).
  • Replies route to sound.otherworld@gmail.com via Cloudflare Email Routing.

Workflow

  • Generate unique ekthesis token per contact at /internal/outreach
  • Send personalized pitch email with token link via Resend
  • Track status: pending → sent → opened/bounced
  • Revoke tokens as needed
Practical workflow: Go to /internal/outreach. Select a contact. Click "Generate Token" to create their unique ekthesis link. Review or customize the pitch email. Click "Send." The system sends the email via Resend from the appropriate address (artist or imprint based on tier), and the contact's status updates to "sent." If they open the ekthesis link, we can update their status to "opened." If the email bounces, mark as "bounced." If a contact should lose access, revoke their token from /internal/tokens.

Clipping Strategy — TikTok / Reels

What clipping is: A paid content-seeding strategy where editors post short clips across multiple accounts in a coordinated campaign — manufacturing organic-feeling discovery without the artist generating new content. What began in livestreaming is now standard practice across major and indie music releases as of 2025–26. For SINE NOCTIS, clipping runs as a parallel awareness layer alongside press and IG, pushing the album's world into algorithmic circulation on TikTok and Reels.

Why It Fits

Clipping works best when footage creates intrigue without demanding explanation. SINE NOCTIS is built for this — the Blanco shoot, BMW interior footage, and SA film stills are atmosphere-first content. Visually complete. Context-independent. Curiosity-generating by design. The short-film framing means clips don’t need to explain who FREE is. They need to make someone stop scrolling.

Clippable Assets

  • Blanco shoot footage — cinematic, monochrome, SN era
  • BMW interior sequences — The Chariot
  • SA film stills — motion export
  • Audio-reactive title card clips
  • Album cover reveal moments

Target Platforms

  • TikTok — primary
  • Instagram Reels

  • X / Twitter — secondary

Campaign Brief — Content Rules

This is the quality control layer. OWJV’s world-building depends on aesthetic discipline. Clippers operate inside tight parameters — this is what separates seeding from flooding.
  • 01 — No memes. Content must be cinematic or atmospheric. No comedy edits, trending audio overlays, or meme formats.
  • 02 — Audio must be SINE NOCTIS. No mixing with trending sounds. The album audio anchors every clip.
  • 03 — On-screen text: minimal or none. If used — Outfit Medium, white, low opacity. No bold overlays or reaction text.
  • 04 — Captions must be discovery-framed. Approved: “who is this,” “what is this,” “this sounds like a film score.” No promotional language.
  • 05 — Tag @freenotavailable or @otherworldlyjustvisiting in every post.
  • 06 — No AI-generated visuals. All footage from the approved OWJV asset library only.
  • 07 — Quality over volume. Fewer high-quality clips outperform bulk posting for this identity. This is not a shitposting campaign.

Rollout Timing

Clipping is a launch-week amplifier, not an announcement tool. Let press and IG establish the world first. Activate the week of release to sustain algorithmic momentum once the album exists on streaming.
  • Pre-Release — IG announcement carousel, press outreach (Patu first), atnocost.cc/EPK live. Clipping: not active.
  • Release Week — Activate — Album live on DSPs. Clipping campaign live (7 days). Reels/TikTok organic posts. Streaming pitch active.

The aesthetic principle: This site is dark, cinematic, and interactive — but not minimal for minimal's sake. Every element has weight and purpose. If something looks empty, it's intentional negative space. If something looks dense, it's earned. The site should feel like entering a physical space, not browsing a webpage. Don't clean things up that are meant to feel textured. Don't simplify things that are meant to feel layered.
Era-based overhauls: The site is not permanently fixed. When a new major project drops (e.g., OTHERLAND), the site may undergo a thematic overhaul to reflect that era's aesthetic. The homepage takes the most change — hero imagery, color palette, photography, and cinematography will shift to match the new release. Existing act sections (FINExME, SINE NOCTIS) are compartmentalized, not removed. They stay accessible but the homepage and top-level experience refocuses on the current era. Think of the site as a living space that redresses itself with each chapter. The bones stay. The skin changes.
