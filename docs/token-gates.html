Token Gate System

atnocost.cc · Updated March 2026

Overview

The site uses token-gated access for private content. Visitors need a valid token appended to the URL to view protected pages. Without one, they see a gate page.

GateURLPurpose
Ekthesis atnocost.cc/ekthesis?token=XXXX Business case / investor deck
Brief atnocost.cc/brief?token=XXXX Creative brief (future use)

How Token Validation Works

  1. Visitor hits /ekthesis or /brief
  2. Middleware checks the ?token= query parameter
  3. Token is validated against Vercel KV (Redis) first, then environment variables as fallback
  4. Valid token → content loads. Invalid or missing → gate page

Token Generator — Admin UI

URL: https://atnocost.cc/internal/tokens

Logging In

  1. Navigate to atnocost.cc/internal/tokens
  2. You'll be redirected to the login page
  3. Enter your password (the value of INTERNAL_AUTH_TOKEN)
  4. Once authenticated, the token dashboard loads

Generating Tokens

  • Select a gate — toggle between ekthesis and brief
  • Enter contact name — e.g., "Alphonse Pierre" (for your reference only)
  • Click Generate — creates a unique 12-character token
  • Copy URL — copies the full shareable link to your clipboard
  • Revoke — permanently deletes the token; the link stops working immediately

Shareable URL Format

https://atnocost.cc/ekthesis?token=abc123def456
https://atnocost.cc/brief?token=abc123def456

Internal Password Setup

The /internal section (token generator, future admin tools) is protected by a single password stored as an environment variable in Vercel.

Setting or Changing the Password

  1. Go to Vercel Dashboard → Project → Settings → Environment Variables
  2. Set INTERNAL_AUTH_TOKEN to your chosen password
  3. Apply to Production, Preview, and Development
  4. Redeploy: npx vercel deploy --prod
Whatever value you set for INTERNAL_AUTH_TOKEN is the password you enter on the login page.

Environment Variables

VariablePurposeRequired
INTERNAL_AUTH_TOKEN Password for /internal admin pages Yes
KV_REST_API_URL Vercel KV Redis endpoint (auto-set when KV is linked) Yes
KV_REST_API_TOKEN Vercel KV auth token (auto-set when KV is linked) Yes
EKTHESIS_TOKENS Comma-separated fallback tokens Optional
BRIEF_TOKENS Comma-separated fallback tokens Optional

Fallback Tokens

Static tokens set via EKTHESIS_TOKENS or BRIEF_TOKENS always work alongside KV-generated tokens. These don't appear in the admin UI and can't be revoked from it — they're managed directly in Vercel's environment variable settings.

Technical Architecture

Visitor → middleware.ts
              |— Check ?token= param
              |— Validate against env vars (instant)
              |— Validate against Vercel KV via REST API
              |— Rewrite to /gate if invalid

Admin → /internal/tokens (React UI)
              |— GET  /api/tokens?gate=ekthesis  → list tokens
              |— POST /api/tokens {gate, contact} → create token
              |— DELETE /api/tokens {gate, token} → revoke token

Storage: Vercel KV (Redis)
  Key: gate:ekthesis  → Hash of { token: JSON metadata }
  Key: gate:brief     → Hash of { token: JSON metadata }

Quick Start

  1. Ensure INTERNAL_AUTH_TOKEN is set in Vercel env vars
  2. Go to atnocost.cc/internal/tokens
  3. Log in with your password
  4. Select gate, enter contact name, click Generate
  5. Share the copied URL with your contact
  6. Revoke anytime to kill access instantly
