Press Outreach & Email System

atnocost.cc · Updated March 2026

Overview

The outreach system sends press emails directly from atnocost.cc/internal/outreach. Emails are delivered via Resend (SMTP relay) from @atnocost.cc addresses. No copy-paste required — click Send, confirm, delivered.

You (outreach tool) → Resend API → Recipient inbox
Recipient replies → Cloudflare Email Routing → Gmail (sound.otherworld@gmail.com)

How to Send

  1. Go to atnocost.cc/internal/outreach
  2. Find or search for a contact
  3. Click to expand — review From, To, Subject, Body
  4. Click Send
  5. Confirmation modal shows the full email — review one more time
  6. Click Send Email to deliver
  7. Status auto-updates to "sent"
Copy buttons are still available on every contact if you ever need to send manually through Gmail instead.

Updating Contacts

To add, remove, or edit a contact: request the change (e.g., "add X from Y to tier Z with email W"). One file edit + redeploy. No database migration, no complex process.

Ekthesis Token Attachment

By default, all emails link to the public EPK at atnocost.cc/epk. For specific contacts who need the full business case:

  1. Expand the contact in the outreach tool
  2. Click Attach Ekthesis
  3. A unique token is generated and the ekthesis link is added to the email body
  4. Send as normal — the link is included automatically
  5. Revoke anytime to kill that contact's ekthesis access

Ekthesis tokens generated here also appear in the standalone Token Generator at /internal/tokens.

Infrastructure

Email Delivery: Resend

Emails are sent via the Resend API. Domain atnocost.cc is verified with DKIM/SPF records in Cloudflare DNS.

Resend API Key: stored as RESEND_API_KEY in Vercel env vars. If the key is rotated, update it there and redeploy.

Email Receiving: Cloudflare Email Routing

Inbound mail to @atnocost.cc is forwarded to sound.otherworld@gmail.com via Cloudflare Email Routing. Replies to outreach emails land here.

Gmail Alias (Backup)

free@atnocost.cc is also configured as a "Send mail as" alias in Gmail using Resend SMTP. This allows manual sending from Gmail if the outreach tool is unavailable.

Troubleshooting

Send button says "RESEND_API_KEY not configured"

Add the API key in Vercel Dashboard → Settings → Environment Variables → redeploy.

Email lands in spam

Check that DKIM and SPF records are present in Cloudflare DNS (added during Resend domain verification). Resend dashboard → Domains → should show green "Verified."

Replies not arriving

Check Cloudflare Email Routing is active and forwarding to the correct Gmail address.

Can't log in to /internal

Verify INTERNAL_AUTH_TOKEN is set in Vercel env vars. The value you set is the password.
